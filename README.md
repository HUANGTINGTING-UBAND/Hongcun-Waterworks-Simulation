# 🎮 宏村水脉：水利演算 (Hongcun Waterworks Simulation)

> **基于安徽宏村真实史实与“牛形水系”的数据驱动水利演算沙盒游戏**

---

## 📌 项目概述

《宏村水脉：水利演算》是一款融合了历史策略与流体网络模拟的独立游戏。玩家在游戏中扮演明清时期徽州宏村的汪氏族长特使，不进行天马行空的自由建造，而是遵照真实历史高程测绘与规划蓝图，通过合理**铺设和疏浚水圳（水渠）**、**调控闸门分流**以及**颁布传统徽州村规民约**，在干旱、梅雨大洪、火灾和人口增长带来的污染压力下，守护宏村数百年水脉系统的动态平衡。

游戏的核心模拟底座基于宏村著名的**“牛形水系”**进行具象化还原：
*   **雷岗山（牛头）**：引西溪天然水源之始。
*   **水圳（牛肠）**：蜿蜒 1200 余米，绕过家家户户的明暗渠水道。
*   **月沼（牛胃）**：位于村中心的半月形池塘，提供中继容积缓冲与泥沙沉淀自净。
*   **南湖（牛肚）**：村南开凿的百亩大湖，用于蓄洪、深度灌溉与最终排水外泄。

## 🌊 核心机制说明

项目采用**无固定网格（Non-Grid）的图论有向图（Directed Graph）拓扑结构**来完美支撑水利学演算，避免传统模拟游戏中网格流体的高额性能开销：

1.  **图论有向图流量模型 (Directed Graph Flow Model)**
    *   水网由**节点（WaterNode）**和**连接边（WaterEdge）**组成。
    *   节点具有物理海拔高度（Elevation）、当前水量（Volume）、最大容量上限（Capacity）以及实时水质评分（Quality）。
    *   连接边具有阀门开度比率（GateOpenRatio）和最大通流上限。流量根据节点间的**高程差势能（Delta H）**进行每帧实时传导演算，完全还原“水往低处流”的自然规律。
2.  **水质动态演化与“晨饮暮洗” (Water Quality & Historic Rhythms)**
    *   每个生活节点会产生本地污染率（LocalPollutionRate）。
    *   **晨饮暮洗令**：还原“上段饮水、中段洗涤、下段灌溉”的族规。清晨时段全村只准取水饮用，污染率降到极低；傍晚洗涤时段，大量洗涤废水汇入，水质将跌破安全阈值 40。
    *   水质受上游注入水质与本地残留水量进行实时体积加权混合计算，并在月沼等静态大水体中具备缓慢的自净能力。
3.  **水渠淤积与生态防堵 (Siltation & Turtle Ecosystem)**
    *   水流过缓、或泥沙含量过高时，水渠会产生淤泥（SiltLevel），从而导致最大通流上限（maxFlowRate）永久缩减，增加上游漫溢风险。
    *   玩家可以通过拨付“白银”雇佣村民人工清淤，或在水道交汇点**“投放乌龟（River Turtles）”**，通过建立暗渠微生态来持续自净化，吞噬淤泥杂物，防止水道堵塞。

## 🗺️ 历史关卡规划大纲 (Campaign Roadmap)

### 🎬 【序章：烈火与干旱的记忆】
*   **史实背景**：南宋至明初，汪氏族人屡受火灾、干旱之苦，宗祠多次毁于大火。
*   **玩法表现**：作为教学互动序幕，体验在无水利引导、无村规制约状态下，宏村遭遇毁灭性旱灾与祠堂“火煞”入侵的危机，激发治水决心。

### 🔨 【第一卷：蓝图与奠基（明永乐年间）】
*   **何可达的沙盘（规划）**：邀请风水测绘大师何可达，利用自然坡度进行高程测绘，确立以雷岗山为牛头、房屋为牛身、村中心凹地为牛胃（月沼）的“牛形蓝图”。
*   **开凿月沼（牛胃）**：由族长汪思齐之妻**胡重**主持，在村中心“穴眼”开凿半月形池塘，实现【容积缓冲】与【沉淀自净】。
*   **开凿水圳（牛肠）**：引水入村。通过拓扑距离强制划分“上饮、中洗、下灌”，并能通过升级“暗渠盖板”降低人为丢弃污染。

### 🌧️ 【第二卷：全貌与圆满（明万历三十五年）】
*   **集资开凿南湖（牛肚）**：万历三十五年（1607年）人口激增，月沼不堪重负，溢出漫溢。全族集资在村南开凿百亩南湖，形成“入水口 → 水圳 → 月沼 → 水圳 → 南湖 → 外排奇墅湖”的闭环。
*   **终极防汛大考**：应对“百年一遇梅雨大洪”。玩家需要全面调控水坝闸门、利用月沼与南湖实行错峰蓄洪、开闸泄洪。

### ♾️ 【第三卷：无尽的守护（沙盒与现代传承）】
*   **机制**：在无尽的时间线里，面对随机发生的百年大旱、连绵阴雨、泥沙淤堵、村规冲突。核心侧重于“精细化村规面板操作”与“自净化养殖生态的维护”。
*   **通关尾声**：通关后播放清代水法修缮史，以及1997年至今宏村申遗成功的现代高清纪录片视频，彰显天人合一、人地和谐的农耕智慧。

## 🛠️ 当前代码架构导航

当前项目底座已实现完整的流体演算引擎及村规维护系统，核心代码结构如下：

```
Assets/Scripts/
├── WaterNetworkManager.cs     # 底层流体有向图仿真引擎。每帧负责计算高程流速、水质加权混合及漫溢洪水警告。
├── VillagePolicyManager.cs    # 传统规约与维护管理器。负责清淤、养龟微生态计算及晨饮暮洗令对排污率的修正。
└── WaterLineRenderer.cs       # 表现层特效组件。每帧自动读取水流边数据，自适应计算线条宽度，并根据水质进行亮蓝-浑浊褐色的渐变插值，通过材质平移模拟真实流速动画。
```

### 🤝 协同运行逻辑
1. `WaterNetworkManager` 在主循环中每帧执行 `SimulationTick`。
2. 每一个代表水渠的 `GameObject` 挂载 `WaterLineRenderer`，通过 `Bind(WaterEdge)` 或者节点名称联结对应的拓扑边。
3. `WaterLineRenderer` 在 `Update` 中自动计算：
   $$\text{Width} = \text{Mathf.Lerp}(\text{minWidth}, \text{maxWidth}, \frac{\text{flowRate}}{\text{maxFlowReference}})$$
   并通过对单点水质评估的颜色进行插值（优质亮蓝 $\ge 70 \to$ 跌落灰蓝 $\ge 40 \to$ 污染灰褐），最终实时刷新 Unity 的 `LineRenderer` 和贴图流动速度。
4. `VillagePolicyManager` 结合当前时段和政策状态，在每帧修正节点的 `localPollutionRate` 及边的 `maxFlowRate`。

## 🚀 快速集成说明

1.  将 `WaterLineRenderer.cs`，`WaterNetworkManager.cs`，`VillagePolicyManager.cs` 放置于 Unity 项目的 `Assets/Scripts/` 目录下。
2.  在场景中创建一个空 GameObject 并挂载 `WaterNetworkManager` 作为整个系统的管理中枢。
3.  构建水渠（WaterEdge）的表现层 GameObject，挂载 `LineRenderer` 和 `WaterLineRenderer` 组件。
4.  确保 LineRenderer 材质的 **Shader** 支持主贴图偏移，且贴图的 **Wrap Mode** 已设置为 **Repeat**，以便呈现美观的流动波纹动画。
