# 宏村水脉 Project Bible v2.1

# VISUAL BIBLE 02：Asset Pipeline 资产生产规范

版本： Project Bible v2.1

文档类型： Visual Production Guideline

用途：

定义《宏村水脉》的数字资产来源、制作流程、格式规范和质量标准。

------------------------------------------------------------------------

# 1. 资产总体原则

《宏村水脉》不是简单模型拼接，而是建立统一的数字遗产资产库。

所有资产必须服务：

-   历史真实性；
-   水系演化；
-   聚落空间；
-   玩家互动体验。

核心流程：

    参考资料

    ↓

    资产制作

    ↓

    Blender

    ↓

    GLB模型

    ↓

    React Three Fiber加载

    ↓

    网页互动

------------------------------------------------------------------------

# 2. 资产来源分级

## A级：真实资料资产

优先来源：

-   官方测绘资料；
-   GIS数据；
-   航拍资料；
-   学术研究资料。

主要用于：

-   地形；
-   水系；
-   空间关系。

------------------------------------------------------------------------

## B级：专业商业资产

来源：

-   Unreal Marketplace；
-   Sketchfab；
-   TurboSquid；
-   CGTrader。

适用于：

-   古建筑；
-   环境；
-   植被；
-   道具。

要求：

-   检查授权；
-   检查格式；
-   检查面数。

------------------------------------------------------------------------

## C级：AI辅助资产

用于：

-   概念设计；
-   快速原型；
-   非核心装饰。

禁止直接决定：

-   核心历史建筑；
-   水利结构；
-   历史人物形象。

------------------------------------------------------------------------

------------------------------------------------------------------------

# 5. 建筑资产标准

采用三级LOD。

## LOD3 远景

用于：

宏村整体地图。

要求：

-   白墙；
-   黑瓦；
-   建筑轮廓。

------------------------------------------------------------------------

## LOD2 中景

用于：

村庄浏览。

增加：

-   马头墙；
-   门窗；
-   庭院。

------------------------------------------------------------------------

## LOD1 近景

用于：

任务交互。

增加：

-   木结构；
-   石材；
-   水井；
-   装饰细节。

------------------------------------------------------------------------

# 6. 水体资产规范

## 西溪

自然河流。

表现：

-   曲线；
-   流动；
-   河岸生态。

------------------------------------------------------------------------

## 水圳

人工水系。

表现：

-   狭窄；
-   明确边界；
-   缓慢流动。

------------------------------------------------------------------------

## 月沼

核心水景。

表现：

-   平静水面；
-   建筑倒影；
-   公共空间感。

------------------------------------------------------------------------

## 南湖

大型水体。

表现：

-   开阔；
-   蓄水空间；
-   景观节点。

------------------------------------------------------------------------

# 7. 性能标准

网页端目标：

单模型：

    <100k triangles

纹理：

    1K~2K

格式：

    GLB

优化：

-   Draco压缩；
-   LOD；
-   贴图合并。

# 9. Codex资产规则

AI生成代码时：

必须：

1.  不假设不存在的模型；
2.  使用asset manifest；
3.  保留placeholder；
4.  等待真实资产替换。

------------------------------------------------------------------------

# 10. 第一阶段资产目标

Vertical Slice只需要：

    terrain/

    - 雷岗山
    - 西溪


    water/

    - 自然溪流


    buildings/

    - 基础徽派建筑块


    environment/

    - 雾
    - 植被

暂不制作：

-   全村建筑；
-   完整NPC模型；
-   所有装饰细节。

------------------------------------------------------------------------

# 11. 总结

代码负责：

让世界运行。

资产负责：

让世界可信。

《宏村水脉》的视觉目标：

不是复制一个旅游景点，而是建立一个可以理解水、土地和人的关系的数字遗产世界。
