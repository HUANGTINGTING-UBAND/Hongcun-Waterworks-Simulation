# 宏村水脉 Project Bible v2.1

# VISUAL BIBLE 04：UI / UX / Interaction Design

版本： Project Bible v2.1

文档类型： Visual Interaction Guideline

用途：

定义《宏村水脉》的用户界面、交互方式、信息展示和玩家体验流程规范。

------------------------------------------------------------------------

# 1. UI设计核心定位

《宏村水脉》不是传统建造游戏HUD。

UI目标：

让用户像阅读一幅会生长的历史地图一样理解宏村。

核心：

    观察

    ↓

    理解

    ↓

    设计

    ↓

    决策

    ↓

    改变世界

------------------------------------------------------------------------

# 2. 首页设计

用户进入游戏：

不是传统菜单。

而是一张宏村最终状态数字地图。

展示：

-   雷岗山；
-   西溪；
-   水圳；
-   月沼；
-   南湖；
-   古村空间关系。

入口：

    进入宏村历史

开始时间旅程。

------------------------------------------------------------------------

# 3. 主界面布局

## 中央区域

3D世界。

占主要视觉空间。

用于：

-   浏览地图；
-   观察水系；
-   查看建筑。

------------------------------------------------------------------------

## 左侧信息面板

展示：

-   历史阶段；
-   地理背景；
-   工程记录。

示例：

    永乐时期

    水利建设阶段

    开始规划引水系统

------------------------------------------------------------------------

## 右侧任务面板

展示：

-   当前任务；
-   目标；
-   完成条件。

示例：

    任务：

    完成地形勘测

    目标：

    确定引水位置

------------------------------------------------------------------------

## 底部时间轴

展示：

    自然溪流

    ↓

    水圳建设

    ↓

    月沼形成

    ↓

    南湖扩建

    ↓

    现代宏村

------------------------------------------------------------------------

# 4. 地图交互

支持：

## 缩放

理解：

村庄整体结构。

------------------------------------------------------------------------

## 旋转

观察：

-   山水关系；
-   水流方向；
-   建筑布局。

------------------------------------------------------------------------

## 点击

对象：

-   山；
-   水；
-   建筑；
-   NPC；
-   工程节点。

------------------------------------------------------------------------

# 5. 水利设计交互

核心玩法：

玩家参与工程设计。

不是简单点击按钮。

流程：

    选择位置

    ↓

    生成设计节点

    ↓

    调整路线

    ↓

    模拟结果

    ↓

    评价方案

系统反馈：

-   水流；
-   覆盖范围；
-   维护成本；
-   社会影响。

------------------------------------------------------------------------

# 6. 信息展示方式

避免：

大量文字堆积。

采用：

## 信息卡片

例如：

    月沼

    功能：

    蓄水
    调节
    公共空间

    阶段：

    水利建设完成后形成

------------------------------------------------------------------------

## 数据图示

例如：

    西溪

    ↓

    堨坝

    ↓

    水圳

    ↓

    月沼

------------------------------------------------------------------------

# 7. NPC交互设计

NPC不采用传统：

"头顶感叹号"。

采用环境叙事。

方式：

玩家接近：

出现：

-   工程记录；
-   历史对话；
-   任务提示。

------------------------------------------------------------------------

# 8. 时间轴交互

玩家可以：

拖动历史时间。

例如：

    1400

    ↓

    1500

    ↓

    1600

    ↓

    现代

地图同步变化：

-   水系；
-   建筑；
-   人口；
-   社会状态。

------------------------------------------------------------------------

# 9. 学习系统

奖励不是金币。

奖励：

知识解锁。

包括：

-   宏村水规；
-   徽州建筑知识；
-   水利工程记录；
-   历史资料。

------------------------------------------------------------------------

# 10. 纪录片入口

最终章节：

完成全部工程后。

进入：

历史资料学习模式。

流程：

    游戏体验

    ↓

    历史理解

    ↓

    官方资料学习

------------------------------------------------------------------------

# 11. UI视觉规范

颜色：

    宣纸白

    徽墨黑

    黛青

    水绿色

    土黄色

风格：

-   东方；
-   克制；
-   清雅；
-   数字博物馆感。

避免：

-   科幻界面；
-   高饱和游戏HUD。

------------------------------------------------------------------------

# 12. 移动端适配

支持：

-   桌面浏览器；
-   平板；
-   手机。

优先交互：

-   点击；
-   拖动；
-   缩放。

------------------------------------------------------------------------

# 13. Vertical Slice UI目标

第一阶段必须完成：

✅ 首页宏村地图

✅ 时间轴

✅ 基础任务卡

✅ 信息弹窗

✅ 地图点击交互

------------------------------------------------------------------------

# 14. 总结

UI的目标：

不是让玩家管理数字。

而是让玩家理解：

水如何塑造土地，

土地如何塑造村庄，

村庄如何形成文化。

END
