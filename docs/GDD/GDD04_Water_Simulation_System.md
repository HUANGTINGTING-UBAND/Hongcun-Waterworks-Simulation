# 1. 系统目标

水利系统是《宏村水脉》的核心玩法系统。

游戏重点不是建造建筑，而是理解：

自然水文条件如何通过工程技术和社会组织，形成长期运行的村庄生命系统。

核心逻辑：

自然水源

↓

引水工程

↓

水圳网络

↓

月沼调蓄

↓

南湖扩展

↓

居民生活与农业

---

# 2. 水系统设计原则

## 原则一：水必须服从地形

水流由：

- 高程；
- 坡度；
- 地形阻挡；
- 工程设施

共同决定。

禁止：

无工程情况下逆地形流动。

---

## 原则二：工程改变水，但不能违反水文规律

玩家可以：

- 修筑堨坝；
- 调整水圳；
- 设置闸门；
- 建设蓄水空间。

但所有结果必须符合模拟规则。

---

# 3. 水利拓扑模型

采用：

Node - Edge 水网络模型。


结构：

```
水源

↓

引水节点

↓

堨坝

↓

主水圳

↓

支流水圳

↓

月沼

↓

南湖

↓

农田
```

---

# 4. 水节点 Water Node

每个节点包含：

```json
{
"id":"moon_pond",
"type":"reservoir",
"elevation":120,
"capacity":500
}
```

字段：

id：

唯一编号。

type：

节点类型。

elevation：

高度。

capacity：

储水能力。

---

节点类型：

- river
- dam
- gate
- canal
- pond
- lake

---

# 5. 水流模型

核心：

重力自流。


基本规则：

```
水流方向 = 高程差决定
```

影响因素：

- 水位差；
- 渠道长度；
- 渠道宽度；
- 摩擦阻力；
- 闸门状态；
- 降雨量。

---

水路数据：

```json
{
"from":"xixi",
"to":"canal001",
"slope":0.003
}
```

---

# 6. 水圳系统

水圳不是普通水渠。

它承担：

- 输水；
- 降速；
- 沉淀；
- 分配。

---

九曲十弯设计逻辑：

- 适应地形；
- 降低流速；
- 增加净化过程；
- 服务不同居民需求。

---

错误设计：

水流过快：

↓

冲刷。


水流不足：

↓

无法供水。

---

# 7. 月沼系统

月沼不是装饰水景。

功能：

- 蓄水；
- 调节流量；
- 改善环境；
- 支撑居民生活。


数据：

```json
{
"type":"pond",
"storage":1000,
"waterQuality":90
}
```

---

# 8. 南湖系统

南湖属于后期公共工程。

解锁条件：

- 人口增长；
- 经济积累；
- 水系统压力增加。

作用：

- 调蓄；
- 防洪；
- 景观；
- 公共空间。

---

# 9. 水质系统

水质随着水流过程变化。


模拟：

```
上游

↓

流动

↓

沉淀

↓

净化

↓

居民使用
```

影响因素：

- 流速；
- 水圳长度；
- 污染来源；
- 清淤频率。

---

# 10. 清淤维护系统

工程完成不是终点。

玩家必须：

- 安排劳动力；
- 定期维护；
- 修复设施。


维护失败：

↓

水量下降

↓

居民满意度降低

↓

村庄发展受阻。

---

# 11. 灾害系统

## 洪水

触发：

暴雨事件。


检测：

- 水圳容量；
- 月沼储量；
- 南湖状态；
- 排洪能力。


结果：

设计合理：

↓

降低灾害影响。


设计错误：

↓

村庄受损。

---

## 火灾

水系统同时承担消防功能。

检测：

- 水源距离；
- 储水能力；
- 消防路径。


---

# 12. 居民用水规则

居民需求：

- 饮水；
- 洗涤；
- 灌溉；
- 防火。


形成：

水规系统。


玩家需要协调：

个人需求

与

公共水资源。

# 17. 设计总结

宏村水利系统的核心不是“有水”。

而是：

如何让水按照自然规律运行，并通过工程和制度服务一个村庄数百年。
