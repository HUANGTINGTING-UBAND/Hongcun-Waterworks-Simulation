# 宏村水脉

# Hongcun Water Pulse

# GDD02_Historical_Map_System.md

版本： Project Bible v2.1

文档类型： Game Design Document

用途：

定义《宏村水脉》的历史地图系统、地形演化、水系演变以及GIS/3D开发规范。

------------------------------------------------------------------------

# 1. 系统目标

历史地图系统是整个游戏世界的基础。

目标：

让玩家理解：

宏村不是先有建筑，而是在特定山水环境、水文条件和工程技术作用下逐渐形成。

地图系统需要表现：

-   自然地形；
-   河流水系；
-   工程改变；
-   聚落扩展；
-   历史时间变化。

------------------------------------------------------------------------

# 2. 地图设计原则

## 原则一：先自然，后人工

地图演化顺序：

自然地貌

↓

自然溪流

↓

人工引水

↓

水圳网络

↓

月沼形成

↓

南湖扩展

↓

成熟聚落

------------------------------------------------------------------------

## 原则二：地形决定水流

所有水流必须遵循：

-   高程；
-   坡度；
-   地形阻挡；
-   工程设施。

禁止：

无工程情况下水逆地势流动。

------------------------------------------------------------------------

# 3. 历史地图阶段

## Stage 0：自然环境阶段

时间：

建村之前。

地图内容：

-   雷岗山；
-   河谷；
-   原始溪流；
-   冲积地形。

玩家目标：

理解宏村形成的自然基础。

------------------------------------------------------------------------

# Stage 1：水系演变阶段

内容：

自然河道逐渐稳定。

重点表现：

羊栈溪、羊栈河、西溪等水系关系。

说明：

具体演变路径需要依据历史研究资料建立。

数据等级：

B级：

多来源研究。

------------------------------------------------------------------------

# Stage 2：人工水利阶段

玩家参与：

-   地形测绘；
-   引水点选择；
-   堨坝建设；
-   水圳规划。

核心逻辑：

自然水源

↓

工程调节

↓

# Stage 3：成熟宏村阶段

# 6. 地形数据规范

示例：

``` json
{
"id":"leigang",
"type":"mountain",
"elevation":220,
"slope":35,
"confidence":"A"
}
```

字段：

id：

唯一编号。

type：

地貌类型。

elevation：

高程。

confidence：

历史可信等级。

------------------------------------------------------------------------

# 7. 水系拓扑模型

水系统采用节点-边模型。

Node：

表示：

-   河流节点；
-   堨坝；
-   水圳入口；
-   月沼；
-   南湖。

Edge：

表示：

-   水流连接；
-   渠道关系。

示例：

``` json
{
"from":"xixi",
"to":"canal001",
"slope":0.003
}
```

------------------------------------------------------------------------

# 8. 玩家地图操作

玩家可以：

## 查看

-   地形；
-   高程；
-   水流方向。

## 测量

-   距离；
-   坡度；
-   水位差。

## 规划

-   引水位置；
-   工程路线。

## 验证

观察：

-   水是否流通；
-   村庄是否受益；
-   灾害风险是否降低。

# 12. 设计总结

地图不是背景。

地图是宏村水利文明形成的第一因。

玩家首先认识土地，

然后理解水，

最后理解村庄。
