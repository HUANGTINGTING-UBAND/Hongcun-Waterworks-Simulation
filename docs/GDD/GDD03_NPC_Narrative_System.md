# 宏村水脉

# Hongcun Water Pulse

# GDD03_NPC_Narrative_System.md

版本： Project Bible v2.1

文档类型： Game Design Document

用途：

定义《宏村水脉》中的NPC系统、历史叙事结构、角色功能以及NPC与任务系统的连接方式。

------------------------------------------------------------------------

# 1. NPC系统目标

NPC不是为了制造传奇故事。

NPC的作用：

> 作为玩家理解宏村水利体系形成过程的历史接口。

通过不同角色，玩家理解：

-   为什么需要规划；
-   如何组织工程；
-   如何协调村民；
-   如何维护水系统。

------------------------------------------------------------------------

# 2. NPC设计原则

## 原则一：历史角色不是英雄

宏村水利体系不是由单个人创造。

它来自：

自然条件

+

专业知识

+

宗族组织

+

工程技术

+

居民维护

------------------------------------------------------------------------

## 原则二：NPC代表社会功能

  角色             代表功能
  ---------------- ----------------
  汪氏宗族负责人   组织与决策
  何可达           规划与知识
  胡重             工程推进与协调
  工匠群体         技术实施
  村民群体         需求与维护

------------------------------------------------------------------------

# 3. 核心NPC设计

## 3.1 汪氏宗族负责人

类型：

组织型NPC

功能：

-   发起水利规划；
-   组织资源；
-   协调村庄事务。

玩家关系：

第一阶段主要引导角色。

对应系统：

-   Settlement System
-   Quest System

------------------------------------------------------------------------

## 3.2 何可达

类型：

规划辅助NPC。

功能：

带领玩家：

-   观察地形；
-   分析高差；
-   理解水流方向；
-   参与规划。

对应玩法：

地图测绘。

------------------------------------------------------------------------

## 3.3 胡重

类型：

工程组织NPC。

功能：

将规划方案转化为实际工程。

任务：

-   组织劳动力；
-   协调施工；
-   推进水利设施建设。

------------------------------------------------------------------------

## 3.4 工匠群体

类型：

功能NPC。

代表：

古代工程技术。

负责：

-   堨坝施工；
-   水圳建设；
-   桥梁建设；
-   后期维护。

------------------------------------------------------------------------

## 3.5 村民群体

类型：

动态群体NPC。

作用：

提出真实生活需求：

-   饮水；
-   灌溉；
-   防火；
-   住宅扩展；
-   水质维护。

推动玩家优化水系统。

------------------------------------------------------------------------

# 4. NPC数据结构

示例：

``` json
{
"id":"hu_zhong",
"role":"engine_manager",
"historicalConfidence":"B",
"gameRole":"guide",
"availableTasks":[]
}
```

字段：

id：

角色编号。

role：

社会功能。

historicalConfidence：

历史可信等级。

gameRole：

游戏功能。

------------------------------------------------------------------------

# 5. 对话系统设计

采用：

事件驱动对话。

不是单纯文字聊天。

例如：

玩家错误规划水圳：

NPC反馈：

"此处地势不足，水流无法自然进入村中。"

触发：

重新测量任务。

------------------------------------------------------------------------

# 7. 历史可信度管理

所有NPC必须记录：

``` json
{
"name":"Hu Zhong",
"historicalConfidence":"B",
"source":"historical_database",
"gameRole":"guide"
}
```

------------------------------------------------------------------------

# 8. 禁止事项

禁止：

-   添加未经审核历史人物；
-   将传说人物作为事实；
-   让单一NPC完成全部工程；
-   用英雄叙事替代社会协作。

------------------------------------------------------------------------

# 10. Codex任务拆分

## TASK-NPC-001

创建NPC数据系统。

要求：

-   JSON驱动；
-   支持历史等级。

------------------------------------------------------------------------

## TASK-NPC-002

创建事件对话系统。

要求：

-   根据任务状态触发；
-   支持多阶段剧情。

------------------------------------------------------------------------

## TASK-NPC-003

创建NPC任务接口。

要求：

-   NPC可以提供任务；
-   任务状态可保存。

------------------------------------------------------------------------

# 11. 验收标准

完成后：

玩家可以：

-   与NPC互动；
-   获取历史信息；
-   接受任务；
-   推进工程阶段。

系统：

-   数据驱动；
-   支持时间演化；
-   不依赖硬编码剧情。

------------------------------------------------------------------------

# 12. 设计总结

NPC不是故事的中心。

宏村水利系统才是故事的中心。

NPC负责让玩家理解：

人如何认识土地，

如何组织工程，

如何共同维护一个持续数百年的水系统。
