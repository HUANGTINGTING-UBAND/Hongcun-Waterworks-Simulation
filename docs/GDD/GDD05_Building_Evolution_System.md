# 宏村水脉
# Hongcun Water Pulse

# GDD05_Building_Evolution_System.md

版本：
Project Bible v2.1

文档类型：
Game Design Document

用途：

定义《宏村水脉》中的建筑、民居、聚落演化系统。

核心思想：

建筑不是独立放置的装饰资产，而是在地形、水系统、人口需求和社会组织共同作用下逐渐形成的结果。

---

# 1. 系统定位

宏村聚落形成逻辑：

```
自然环境

↓

水利系统

↓

居民需求

↓

建筑布局

↓

聚落演化
```

玩家不能简单点击放置房屋。

玩家需要理解：

为什么建筑会出现在这里。

---

# 2. 建筑生成原则

## 原则一：建筑受环境约束

建筑位置必须考虑：

- 地形稳定性；
- 水源距离；
- 道路连接；
- 公共空间；
- 消防条件。


---

## 原则二：建筑服务生活

建筑需要满足：

- 居住；
- 储水；
- 生产；
- 家族活动；
- 公共交流。


---

# 3. 民居系统

## 3.1 临水住宅

特点：

- 靠近水圳；
- 可直接取水；
- 维护水路便利。


优势：

- 用水方便；
- 消防能力强。


风险：

- 水灾影响增加。


---

## 3.2 非临水住宅

需要：

- 天井；
- 储水设施；
- 雨水收集。


特点：

依靠建筑自身完成水循环。


---

# 4. 天井系统

天井不是视觉装饰。

它是传统建筑中的微型水循环系统。


流程：

```
屋顶降雨

↓

天井收集

↓

家庭储水

↓

生活使用

↓

溢流排放
```


数据：

```json
{
"type":"tianjing",
"storage":500,
"waterConnection":true
}
```

---

# 5. 徽派建筑规则

建筑表现：

- 马头墙；
- 合院；
- 天井；
- 中轴布局。


系统重点：

不是简单复制建筑外观。

而是表现：

建筑如何适应：

- 气候；
- 水环境；
- 家族结构；
- 防火需求。


---

# 6. 聚落扩展系统

人口增长流程：

```
人口增加

↓

住房需求

↓

寻找建设区域

↓

连接水系统

↓

形成新区
```


玩家需要考虑：

- 是否扩大住宅区；
- 是否增加水利设施；
- 是否影响原有水循环。


---

# 7. 建筑对水系统的影响

建筑增长会改变：

- 用水量；
- 排水量；
- 消防需求。


例如：

新增大量住宅：

↓

水需求增加

↓

水圳压力提高

↓

需要：

- 新支圳；
- 新储水设施；
- 新水规。


---

# 8. 消防系统

宏村水系的重要功能之一：

防火。


检测：

- 建筑距离水源；
- 巷道通行；
- 储水能力。


火灾事件：

合理布局：

↓

降低损失。


---

# 9. 建筑数据结构

示例：

```json
{
"id":"house001",
"type":"residence",
"waterAccess":"canal",
"hasTianjing":true,
"yearBuilt":1500
}
```

字段：

id：

建筑编号。

type：

建筑类型。

waterAccess：

水源连接方式。

hasTianjing：

是否具有天井。

yearBuilt：

建造时间。

# 12. 验收标准

完成后：

玩家可以：

- 根据水系规划住宅；
- 建造不同类型民居；
- 设置天井；
- 观察人口增长；
- 处理消防问题。


程序：

- 建筑数据驱动；
- 与水系统连接；
- 支持历史阶段变化。


---

# 13. 设计总结

宏村建筑不是水利系统之后的附属物。

建筑与水共同形成村庄生命系统。

```
地形

↓

水

↓

建筑

↓

人口

↓

社会规则

↓

再次影响水
```

END
