# 3. 核心开发原则

## 3.1 历史真实性优先

禁止：

-   编造历史事件；
-   添加未经审核历史人物；
-   修改宏村历史时间线；
-   将游戏推推演描述为历史事实。

所有历史相关内容必须引用：

    historical_database/

------------------------------------------------------------------------

## 3.2 数据驱动开发

禁止：

将历史逻辑直接写死在代码。

错误：

``` javascript
if(year===1607){
 createSouthLake();
}
```

正确：

读取：

    data/timeline/

例如：

``` json
{
 "year":1607,
 "event":"south_lake_expansion"
}
```

------------------------------------------------------------------------

## 3.3 模块化设计

系统必须独立：

    terrain

    water

    building

    npc

    quest

    economy

    event

禁止：

不同系统互相硬编码依赖。

------------------------------------------------------------------------

# 4. 代码规范

## TypeScript

必须：

-   使用明确类型；
-   避免any；
-   保持接口清晰。

示例：

``` typescript
interface WaterNode {
 id:string;
 type:string;
 elevation:number;
 capacity:number;
}
```

------------------------------------------------------------------------

## 文件命名

组件：

    PascalCase

例如：

    WaterSimulation.tsx

工具：

    camelCase

例如：

    calculateFlow.ts

数据：

    snake_case

例如：

    water_nodes.json

------------------------------------------------------------------------

# 5. React / Three.js规范

## React

要求：

-   组件单一职责；
-   避免巨大组件；
-   状态管理清晰。

------------------------------------------------------------------------

## Three.js / R3F

要求：

3D逻辑与数据分离。

禁止：

直接在模型代码中写：

-   历史数据；
-   水流参数；
-   建筑属性。

正确：

    data

    ↓

    simulation

    ↓

    render

------------------------------------------------------------------------
