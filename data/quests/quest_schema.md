# 宏村水脉 Project Bible v2.1

# DATA_SCHEMA_06_Quest.md

版本： Project Bible v2.1

文档类型： Data Schema Design Document

用途：

定义《宏村水脉》任务系统的数据结构，用于连接历史事件、NPC、玩家操作和世界状态变化。

------------------------------------------------------------------------

# 1. 系统定位

Quest系统负责：

-   历史叙事；
-   玩家引导；
-   工程推进；
-   系统解锁。

核心循环：

    历史事件

    ↓

    NPC需求

    ↓

    玩家任务

    ↓

    玩家决策

    ↓

    世界变化

------------------------------------------------------------------------

# 2. 任务类型

## Main Quest 主线任务

对应宏村水利建设核心过程。

包括：

-   地形勘测；
-   引水规划；
-   堨坝建设；
-   水圳设计；
-   月沼形成；
-   南湖建设。

------------------------------------------------------------------------

## Side Quest 支线任务

用于模拟：

-   村民需求；
-   建筑调整；
-   水资源管理；
-   灾害处理。

------------------------------------------------------------------------

# 3. 主线任务示例

## Q001：寻找理想引水点

阶段：

水利规划阶段。

NPC：

何可达。

目标：

-   测量地形；
-   分析高程；
-   确定水源位置。

结果：

生成水利设计图。

------------------------------------------------------------------------

## Q002：修筑堨坝

目标：

确定：

-   位置；
-   高度；
-   调节能力。

结果：

开启水圳建设。

------------------------------------------------------------------------

## Q003：设计水圳路线

玩家操作：

-   点击节点；
-   绘制路线；
-   调整弯曲程度。

系统检测：

-   高程；
-   坡度；
-   流速；
-   覆盖居民数量；
-   维护成本。

------------------------------------------------------------------------

## Q004：形成月沼

目标：

建设：

-   蓄水空间；
-   调节节点；
-   公共空间。

------------------------------------------------------------------------

## Q005：建设南湖

条件：

-   人口；
-   财富；
-   社会支持；
-   水利需求。

------------------------------------------------------------------------

# 8. 失败反馈

失败不是结束。

例如：

水圳坡度错误：

结果：

-   水流过快；
-   渠道冲刷；
-   居民满意度下降。

玩家可以重新调整方案。

------------------------------------------------------------------------
