# 宏村水脉 Project Bible v2.1

# DATA_SCHEMA_01_Terrain.md

版本：Project Bible v2.1

用途：
定义《宏村水脉》项目中的地形数据库结构，为GIS数据、DEM高程、Blender地形资产以及Three.js场景加载提供统一规范。

------------------------------------------------------------------------

# 1. 系统定位

Terrain系统是整个项目的数据基础。

宏村形成逻辑：

山 → 水 → 村址 → 建筑 → 社会系统

地图不是背景，而是游戏世界运行的基础。

------------------------------------------------------------------------

# 2. Terrain系统负责内容

包括：

-   山体
-   河流
-   河谷
-   高程
-   坡度
-   可建设区域
-   历史阶段变化

------------------------------------------------------------------------

# 3. 数据可信度等级

所有历史数据必须标注可信等级：

A级： 明确测绘数据、公开数据。

B级： 多来源历史研究。

C级： 地方文化解释。

D级： 游戏模拟设计。

------------------------------------------------------------------------

# 4. Terrain基础数据结构

``` json
{
  "id":"object_id",
  "type":"terrain_type",
  "geometry":[],
  "elevation":0,
  "slope":0,
  "confidence":"A"
}
```

------------------------------------------------------------------------

# 5. 山体 Mountain

示例：

``` json
{
"id":"leigang",
"type":"mountain",
"elevation":220,
"slope":35,
"confidence":"A"
}
```

------------------------------------------------------------------------

# 6. 河流 River

示例：

``` json
{
"id":"xixi",
"type":"river",
"path":[
[0,0,10],
[10,0,8]
],
"confidence":"B"
}
```

------------------------------------------------------------------------

# 7. 土地 Land

用于判断：

-   村庄建设
-   农田
-   公共设施

------------------------------------------------------------------------

# 8. 历史阶段数据

同一地点支持多个阶段：

``` json
{
"id":"xixi",
"stages":[
"natural",
"early_settlement",
"water_system"
]
}
```

# 12. 验收标准

系统能够：

-   加载宏村基础地形；
-   显示雷岗山；
-   显示河流路径；
-   计算高低差；
-   为水利模拟提供基础数据。

------------------------------------------------------------------------

# 13. 设计总结

Terrain数据库定义的是：

宏村为什么会出现在这里。

所有后续系统：

水利、建筑、人口、任务

都依赖这一层。
