# 宏村水脉 Project Bible v2.1

# DATA_SCHEMA_04_NPC.md

版本： Project Bible v2.1

文档类型： Data Schema Design Document

用途：

定义《宏村水脉》中的NPC数据结构，用于历史人物、组织角色、村民群体、对话和任务系统。

------------------------------------------------------------------------

# 1. SYSTEM_POSITIONING

NPC系统负责连接：

    历史人物

    ↓

    工程建设

    ↓

    社会组织

    ↓

    玩家任务

    ↓

    村庄演化

NPC不是普通游戏角色，而是历史过程的参与者。

------------------------------------------------------------------------

# 2. NPC分类

## 2.1 历史人物 NPC

来源：

-   地方志；
-   宗族资料；
-   学术研究。

要求：

每个人物必须记录历史可信等级。

------------------------------------------------------------------------

## 胡重

定位：

水利工程组织与实施核心人物。

功能：

-   推动工程实施；
-   协调族人；
-   组织资源。

示例：

``` json
{
"id":"hu_zhong",
"name":"胡重",
"type":"historical_character",
"role":"engineering_manager",
"confidence":"B"
}
```

------------------------------------------------------------------------

## 何可达

定位：

地形规划与水利设计辅助人物。

功能：

-   地形勘测；
-   水路规划；
-   环境分析。

------------------------------------------------------------------------

# 3. 组织型NPC

## 汪氏宗族

不是单个人物，而是社会组织。

功能：

-   组织公共工程；
-   筹集资源；
-   协调村民；
-   制定公共规则。

数据：

``` json
{
"id":"wang_clan",
"type":"organization",
"function":[
"resource_management",
"public_decision"
]
}
```

------------------------------------------------------------------------

# 4. 群体NPC

# 7. 历史可信等级

A级：

明确史料支持。

B级：

多来源研究支持。

C级：

地方文化记忆。

D级：

游戏设计内容。

------------------------------------------------------------------------

# 8. NPC状态系统

示例：

``` json
{
"trust":80,
"support":true,
"available":true
}
```

影响：

-   任务开放；
-   工程支持；
-   社会稳定。

------------------------------------------------------------------------

# 9. NPC与任务关系

结构：

    NPC

    ↓

    Quest

    ↓

    Reward

    ↓

    System Change

例如：

胡重

↓

水利建设任务

↓

工程完成

↓

解锁村庄发展。
