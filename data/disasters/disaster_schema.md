# 宏村水脉 Project Bible v2.1

# DATA_SCHEMA_08_Disaster.md

版本： Project Bible v2.1

文档类型： Data Schema Design Document

用途：

定义《宏村水脉》中的洪水、火灾、风险评估和灾后恢复数据结构。

------------------------------------------------------------------------

# 1. 系统定位

Disaster系统用于模拟：

-   洪水；
-   火灾；
-   极端天气；
-   工程失效；
-   灾后恢复。

核心逻辑：

    自然风险

    ↓

    工程响应

    ↓

    建筑响应

    ↓

    社会恢复

------------------------------------------------------------------------

# 2. 洪水系统 Flood

宏村水利系统的重要价值之一，是降低洪水风险。

影响因素：

-   降雨量；
-   水圳容量；
-   月沼调蓄能力；
-   南湖容量；
-   地形排水条件。

------------------------------------------------------------------------

# 3. 洪水数据结构

示例：

``` json
{
"rainfall":300,
"canalCapacity":100,
"moonPondStorage":200,
"riskLevel":30
}
```

------------------------------------------------------------------------

# 4. 洪水计算模型

基础逻辑：

    洪水风险

    =

    降雨量

    -

    水系统调节能力

    +

    地形风险

------------------------------------------------------------------------

# 5. 水利防洪能力

## 水圳

功能：

-   导流；
-   分散水量。

## 月沼

功能：

-   储水；
-   调节水位。

## 南湖

功能：

-   大规模蓄水；
-   降低洪峰压力。

------------------------------------------------------------------------

# 6. 火灾系统 Fire

宏村传统建筑密集，火灾风险需要模拟。

影响因素：

-   建筑密度；
-   木结构比例；
-   水源距离；
-   道路通行能力。

示例：

``` json
{
"buildingDensity":80,
"waterDistance":20,
"fireRisk":60
}
```

------------------------------------------------------------------------

# 7. 消防响应

流程：

    火灾发生

    ↓

    寻找水源

    ↓

    居民组织

    ↓

    灭火

    ↓

    恢复

影响因素：

-   水圳布局；
-   天井储水；
-   公共水体。

------------------------------------------------------------------------

# 8. 灾后恢复系统

灾害之后：

需要：

-   修复水圳；
-   修复建筑；
-   恢复经济；
-   提升社会稳定。

示例：

``` json
{
"repairCost":100,
"laborNeeded":50,
"time":20
}
```

------------------------------------------------------------------------

# 9. 风险参数

统一风险数据：

``` json
{
"floodRisk":30,
"fireRisk":20,
"maintenanceLevel":90
}
```
